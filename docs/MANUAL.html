FSRIFS (FSR + IFS) Upscale + Frame Interpolator

PRACTICAL USER MANUAL

Hardware Requirements & Dependencies

For the post-processing pipeline to work correctly on your computer, your system must meet the following hardware and software dependencies:

  • Vulkan API Compatibility Required: Your graphics card must have native support for Vulkan technology. It works on dedicated graphics cards (AMD RX, Nvidia GTX/RTX) as well as integrated chips (newer AMD Vega or Intel HD Graphics). If your GPU does not support Vulkan, the process will not start.
  • Processor (CPU): Any basic 4 or 6-core processor (such as the AMD FX line or older generations of Intel Core) is perfectly capable of running the pipeline. Average CPU usage stays around 45%, keeping Windows completely free of stutters or overheating during rendering.
  • Video Memory (VRAM): Legacy graphics cards with 2GB of VRAM will handle the workload smoothly if the user operates intelligently. It is recommended to close web browsers, Discord, and background games before starting the process at high resolutions (such as 2K or 4K) to avoid running out of card memory.
  • Operating System: Windows 10 or Windows 11 64-bit with the execution of PowerShell scripts enabled on the system.

Installation

  1. Start by downloading the zip file containing the pipeline.
  2. Extract the content to your desired directory. You will see the entire folder structure of FSRIFS.
  3. The entire process takes place inside a PowerShell prompt (knowing basic folder navigation commands is important).
  4. Let's start by opening PowerShell; it does not need to be run as administrator since the script does not require elevated privileges to run.

If this is your first time running scripts on your computer, type the command below to allow script execution:

Set-ExecutionPolicy Bypass -Scope Process

Answer YES to apply the execution policy.

How to Execute

Great! Now we are ready to run the processes, and you can do this in two ways:

Method 1: Drag & Drop

This is the easiest way to execute the pipeline. It is tailored for users with no terminal knowledge, allowing you to process an individual video or multiple pre-selected files inside a folder.

  1. Using your file explorer, navigate to the folder where you extracted FSRIFS. You will see a file named DRAG_HERE.bat.
  2. Now open another file explorer window and go to the location of the file or folder you want to process.
  3. Drag your video file or folder and drop it directly onto the DRAG_HERE.bat file.

You can configure the process by editing the DRAG_CONFIG.txt file:

[DRAG_SETTINGS]
QUALITY=med
FPS=60
INTERPOLATE=oversample
SCALE=1920x1080
SHARPNESS=5
VERBOSE=false
SHUTDOWN=false

Method 2: PowerShell (Terminal)

This is the original method to process your videos, requiring basic terminal knowledge.

We need to access the folder where you extracted FSRIFS. For example:

cd "D:\FSRIFS"

You will now be inside the pipeline folder and can execute two scripts. Let's get to know them:

1. process.ps1

Our main script, used to perform upscaling and/or frame interpolation. The script works in a modular way; you can access these modules through command-line parameters:

  • -file "filename.mp4": The full path of the video file you want to process. It must be wrapped in quotation marks.
  • -scale factor|resolution: Final video dimensions. If omitted, upscaling will not occur and the original size is maintained. The value can be provided in the following formats:
    • Multiplier factor for resolution gain. Use decimal numbers (e.g., the scaling factor from a 720p video to 1080p is 1.5).
    • Literal resolution where you can specify, for example:
    • "1280x720" = 720p (HD)
    • "1920x1080" = 1080p (Full HD)
    • "2560x1440" = 2K (Quad HD / QHD)
    • "3840x2160" = 4K (Ultra HD / UHD)
  • -fps number: The desired frame rate for smoothness (e.g., 60). If omitted, frame generation will not occur and the original FPS is maintained.
  • -interpolate "none|oversample|mitchell_clamp|linear": Controls video movement smoothness. If you change the FPS (for example, from 30 to 60), this setting determines how the new frames are generated. The default is "none":
    • none: Disables temporal mixing. Uses pure frame duplication (Nearest Neighbor). Provides maximum GPU performance savings and ensures clean, sharp gaming visuals with absolutely zero ghosting or motion blur.
    • oversample: Enables "Smooth Motion" sampling. Blends frames only when an output frame falls precisely between two source frames. Highly recommended for preserving the original, natural subjective appearance of the video.
    • mitchell_clamp: Utilizes a larger, high-quality filter kernel to smoothly interpolate frames. Designed to eliminate ringing and visual artifacts as much as possible. Best option for maximum fluid smoothness (requires modern GPU/RTX hardware).
    • linear: Applies a standard linear blend/fade between frames. Extremely lightweight temporal processing, ideal for smooth pacing on older hardware configurations.
  • -quality "low|med|big": The profile for final file weight, compression, and size. If you omit this parameter, the system will automatically assume the balanced profile "med". The available options are:
    • low: Focuses on drastic space saving. Reduces the final file size and eases VRAM load. Generates a small final file.
    • med: The ideal sweet spot. Maintains edge sharpness and visual fidelity without blowing up your hard drive storage. Generates a medium-sized final file.
    • big: Maximum visual fidelity and high bitrate, aimed at archiving or high-quality uploads. Generates a huge final file.
  • -sharpness number: Edge crispness applied by FSR, ranging from 0 (minimum) to 10 (maximum). Default is 5.
  • -verbose "false|true": Verbose mode, where detailed information about the processing is printed in the terminal by FFmpeg.
    • false: Displays basic processing data.
    • true: Displays detailed and elaborate processing data.
  • -shutdown "false|true": Controls system power state upon completion. The options are:
    • false: Normal execution behavior. Keeps the computer running.
    • true: Displays a 30-second countdown on the screen. The user can press [ENTER] to turn off immediately, or [ESC] to cancel and keep the PC running.
Safety Note: The script requires you to use at least the -scale or the -fps parameter.

Command Examples:

Perform ONLY Upscaling (e.g., from 720p to 1080p, with medium sharpness 5):

.\process.ps1 -file "C:\path\video.mp4" -scale 1.5 -sharpness 5

Perform ONLY Frame Generation (e.g., your original video is 30fps and you want 60 FPS):

.\process.ps1 -file "C:\path\video.mp4" -fps 60

Full Pipeline (Upscale to 1080p + 60 FPS frame generation + checking a detailed log in the prompt):

.\process.ps1 -file "C:\path\video.mp4" -scale "1920x1080" -fps 60 -verbose true

Output File Naming

The script automatically identifies the origin of your file and saves the final result exactly in the same folder as the input video. The name of the new file is generated intelligently with suffixes based on what you chose to do:

  • If only upscaling was processed: video_FSR_1920x1080.mp4
  • If only frame interpolation was processed: video_IFS_60fpsLINEAR.mp4
  • If the complete pipeline was processed: video_FSR_1920x1080_IFS_60fpsLINEAR.mp4

2. extract.ps1

Script to extract a few frames from the processed video so you can evaluate the result frame by frame. You can extract all frames from X seconds starting at a specific timestamp in the video. It is the ideal tool to visually check the final quality, allowing precise inspection.

Command-line parameters:

  • -file "filename.mp4": The full path of the video file you want to process. It must be wrapped in quotation marks.
  • -time "hh:mm:ss": The starting point for extraction. If left blank, it will automatically start at 3 seconds (00:00:03).
  • -secs number: How many seconds of video you want to convert into images. If left blank, it will extract just 1 second (if the video is at 60fps, then 60 frames will be extracted).
  • -output "destination_path": The folder where images will be saved (wrapped in double quotes). If left blank, the script will extract the frames inside the output folder at the root of FSRIFS.

Known Limitations & Technical Artifacts

Blurring or Smoothing When Paused (Freeze Frame Artifacts)

  • Symptom: When pausing the final video during fast-motion scenes, the image may appear soft, blurry, or show a "ghosting" effect.
  • Cause: The video player froze the playback clock exactly on an intermediate (interpolated) frame generated by the mathematical temporal interpolation engine.
  • Solution: There is nothing to fix. This behavior is the exact mechanism required to guarantee the illusion of smoothness in frame generation without requiring heavy AI or hardware with Tensor Cores. It is the trade-off for a solution targeted at low-end PCs.
