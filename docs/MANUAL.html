FSRIFS (FSR + IFS) Upscale + Frame generator

PRACTICAL USER MANUAL

Hardware Requirements & Dependencies

For the post-processing pipeline to function correctly on your computer, your system must meet the following hardware and software requirements:

  • Vulkan API Compatibility Required: Your graphics card must have native support for Vulkan technology. The custom IFS shader runs temporal calculations directly on the GPU silicon through this API. It works on dedicated graphics cards (AMD RX, Nvidia GTX/RTX) and also on recent integrated chips (AMD Vega or newer Intel HD Graphics). If your GPU does not support Vulkan, the process will not start.
  • Processor (CPU): Any basic older 4 or 6-core processor (such as the AMD FX line or past generations of Intel Core) is perfectly capable of running the pipeline. Average CPU usage stays around 45%, keeping Windows completely free from freezes or overheating during rendering.
  • Video Memory (VRAM): Legacy graphics cards with 2GB of VRAM can handle the task smoothly if the user operates wisely. It is recommended to close web browsers, Discord, and background games before starting the process at high resolutions (such as 2K or 4K) to avoid running out of video memory.
  • Operating System: Windows 10 or Windows 11 (64-bit) with an active PowerShell terminal allowed to execute scripts.

Installation

  1. Start by downloading the ZIP file containing the files.
  2. Extract it wherever you prefer. You will see the entire folder structure of FSRIFS.
  3. The whole process takes place inside a PowerShell prompt (knowing basic folder navigation commands is helpful).
  4. Open PowerShell. You do not need to run it as administrator; the script does not require elevated privileges.

If this is your first time running scripts on this computer, type the command below to allow script execution:

Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process

Answer YES to apply the execution policy.

How to Run

Now you are ready to run the processes. You can do this in two ways:

Method 1: Drag & Drop

This is the easiest way to execute the pipeline. It is tailored for users with no command prompt knowledge.

  1. Using File Explorer, open the folder where you extracted FSRIFS. You will see a file named DRAG_VIDEO_HERE.bat.
  2. Open another File Explorer window and go to the folder where the video you want to process is located.
  3. Drag your video and drop it onto the DRAG_VIDEO_HERE.bat file.

You can configure this process by editing the file DRAG_CONFIG.txt:

[DRAG_SETTINGS]
SCALE=1920x1080
SHARPNESS=5
FPS=60
QUALITY=med
VERBOSE=false

Method 2: PowerShell (Terminal)

This is the original method to process your videos, requiring basic prompt knowledge.

You need to access the directory where you extracted FSRIFS. For example:

cd "D:\FSRIFS"

Once inside the pipeline folder, you can execute two different scripts:

1. process.ps1

Our main script, used to perform upscaling, generate frames, or both. The script is modular; you can access these modules using command-line arguments:

  • -file "filename.mp4": The full path to the video file you want to process. It must be enclosed in quotation marks.
  • -scale factor|resolution: The final target dimension of the video. If omitted, upscaling will not occur, and the original size is kept. The value can be entered as:
    • A multiplier factor for resolution gain using decimal numbers (e.g., the scaling factor from a 720p video to 1080p is 1.5).
    • A literal resolution target, for example: "1920x1080".
  • -fps number: The desired frame rate for fluid motion (e.g., 60). If omitted, frame generation will not occur, and the original FPS is kept.
  • -quality "low|med|big": The profile for weight, compression, and final file size. If omitted, the system defaults to the balanced "med" profile. The options are:
    • low: Focuses on drastic space savings. It reduces the final file size and lightens VRAM usage, generating a small file.
    • med: The ideal sweet spot. It maintains edge sharpness and visual fidelity without overloading storage, generating a medium-sized file.
    • big: Maximum visual fidelity and high bitrate, meant for archiving or high-quality uploads. It generates a huge final file.
  • -sharpness number: The sharpness level applied by FSR, ranging from 0 (minimum) to 10 (maximum).
  • -v: Verbose mode, which lists detailed information about the ongoing process in the terminal.
Safety Note: The script requires you to use at least -scale or -fps.

Command Examples:

Upscale ONLY (720p to 1080p with medium sharpness, using a 1.5x scale and sharpness 5):

.\process.ps1 -file "C:\path\video.mp4" -scale 1.5 -sharpness 5

Frame Generation ONLY (e.g., if your original video is 30fps):

.\process.ps1 -file "C:\path\video.mp4" -fps 60

Full Pipeline (1080p Upscale + 60 FPS frame generation + checking a detailed log in the prompt):

.\process.ps1 -file "C:\path\video.mp4" -scale "1920x1080" -fps 60 -v

Output File Naming

The script identifies the source file and saves the final result directly into the same folder as the original video. The new file name is intelligently generated with suffixes based on your choices:

  • If you only used upscale: video_FSR_1080x720.mp4.
  • If you only used frames: video_60fps.mp4.
  • If you used the full pipeline: video_FSR_1080x720_60fps.mp4.

2. extract.ps1

A script to extract specific frames from the processed video so you can evaluate the quality frame-by-frame. You can extract all frames across a specific number of seconds starting at a chosen timestamp.

Command-line arguments:

  • -file "path": The full path to the video file. Must be enclosed in quotation marks.
  • -time "hh:mm:ss": The starting point for extraction. If left blank, it defaults to 3 seconds (00:00:03).
  • -secs number: How many seconds of video to turn into photos. If left blank, it extracts exactly 1 second (if the video is at 60fps, it will extract 60 frames).
  • -output "destination_folder": Target folder to save the photos (enclosed in double quotes). If left blank, frames are extracted into the output folder inside the FSRIFS root directory.

Known Issues & Technical Limitations

Blurry or Softened Static Frames (Freeze Frame Artifacts)

  • Symptom: When pausing the final video during fast-moving scenes, the image may look soft, blurry, or display a "ghosting" effect.
  • Cause: The video player froze the playback clock exactly on an intermediate (interpolated) frame generated by the temporal calculations of the IFS shader.
  • Solution: There is nothing to fix. This behavior is a technical necessity to guarantee fluid motion when the video is playing without requiring heavy AI or hardware with Tensor Cores. It is the trade-off for a solution designed for low-end PCs.
