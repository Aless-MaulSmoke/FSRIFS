FSRIFS (FSR + IFS) Upscale + Frame generator

PROJECT DESCRIPTION

1. The Project Root (The Need)

Running a demanding game while simultaneously recording in high resolution is a task that requires an extremely robust PC. Even then, these powerful systems struggle to handle this double processing workload in real time. Therefore, the workflow between playing and recording is segmented. First, the game is recorded at a modest resolution so the player can use Ultra graphical settings, including Ray Tracing. After recording, the final video file is submitted to the upscaling process (sharp resolution increase) and sometimes to frame generation.

This way, heavy games originally captured at 30 or 48 FPS receive new artificially created frames, achieving excellent smoothness. However, most market solutions that perform this task depend heavily on intensive AI processing, requiring cutting-edge CPUs and GPUs with dedicated AI cores, as well as massive VRAM resources — requirements completely beyond what low-end graphics cards can supply.

Do you have a low-end PC?
My development environment features an AMD FX-6300 processor and a 4GB Radeon RX 550 GPU. On setups at this tier, it is impossible to run the most popular AI-based upscaling and frame generation solutions on the market. This is exactly where this project becomes necessary: often, the machine's physical limits only allow recording gameplays in 720p at 30fps with good quality, a standard considered very low for current video publishing platforms.

2. The Turning Point: Scaling Resolution and Frames Smoothly

The inspiration for this project arose from practical experience using the Lossless Scaling software in a real-time gaming environment. Observing that a modest PC could successfully run FSR to upscale resolution and use the LSFG frame generator to transform a 720p/30fps gaming experience into a fluid 1080p/60fps gameplay without killing the hardware, a fundamental question emerged:

"If this mathematical principle works so efficiently and smoothly in a real-time game, why can't it be applied to a previously recorded 720p/30fps video file to expand it to 1080p/60fps or more?"

3. A Truly Lightweight Upscale and Frame Generator is Born

To materialize the concept of a lightweight video post-processing pipeline focused on legacy hardware, the power of established open-source tools was combined with custom mathematical logic running directly on the GPU. The ecosystem stands on four fundamental pillars:

ffmpeg + Vulkan + AMD FidelityFX FSR 1.0 + IFS (Intersection Fluid Sharpen)

Although they comprise four distinct technology layers, they all function fused under the same automated process that you easily trigger via commands in your PowerShell:

  • FFmpeg: The core engine responsible for handling, reading, and finally encoding the video streams.
  • Vulkan: The low-level graphics API used as a mandatory dependency. If your graphics card natively supports this technology, hardware acceleration is guaranteed.
  • AMD FidelityFX FSR: A shader focused on spatial upscaling executed directly on your GPU, increasing resolution with edge preservation without relying on any AI infrastructure.
  • IFS (Intersection Fluid Sharpen): A custom shader of my own authorship designed specifically to generate intermediate frames without using AI. It works extremely fast while consuming minimal GPU processing. The inspiration for the shader comes from object intersection processes common in vector graphics software.

3.1. Hardware Requirements & Dependencies

To guarantee stability and proper functionality of the entire post-processing pipeline, your machine must meet the following minimum hardware requirements:

  • Processor (CPU): Any basic or older 4 or 6-core processor (such as the legacy AMD FX line or Intel Core processors from past generations). During the development and validation of this pipeline, an AMD FX-6300 with 8GB RAM (an architecture originally released in 2012) was used.
  • Graphics Card (GPU): It is mandatory for your card to have native support for the Vulkan API. The ecosystem runs perfectly on both dedicated GPUs (AMD RX, Nvidia GTX/RTX) and recent integrated graphics solutions (AMD Vega or newer Intel HD Graphics). In our test lab, we used an entry-level AMD Radeon RX 550 with 4GB VRAM (released in 2017).

Even when operating under these severe physical limitations and utilizing legacy test hardware, the ecosystem demonstrated excellent software efficiency. The pipeline was capable of delivering rendering speeds up to 2.0x (half the video duration time) for output files configured at 1080p/60fps, alongside total stability during heavy 4K rendering processes.

4. Conclusion and Efficiency Verdict

This concept serves as practical proof that intelligent software optimization can bypass severe physical limitations of outdated hardware. Replacing heavy artificial neural network calculations with direct, clean, and cheap mathematical instructions for the GPU silicon architecture brings fresh life to legacy computers. It delivers everything that Lossless Scaling offers, adapted for the local video file post-processing scenario.

"Since the entire upscaling process of this ecosystem is performed via AMD FSR technology, it is highly recommended that your source files consist strictly of video game gameplays. The algorithm was specifically developed to process and polish digital graphics elements, polygon edges, and real-time rendering textures. Attempting to submit real-life recordings — such as movies, series, or videos captured by smartphone cameras — will break the mathematical logic of the process, and you may not achieve the expected result."
